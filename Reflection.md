### Question 1: What are the primary strengths and weaknesses of using AI tools to assist with data science project documentation?
* **Strengths:** AI tools significantly accelerate the initial setup phase by instantly generating clean Markdown formatting, standard document frameworks, and proper hierarchical headings. They also act as an excellent technical editor, identifying potential layout issues and ensuring consistent coding syntax notation across different document types.
* **Weaknesses:** AI tools lack true domain-specific context regarding local file directory setups and unique variables unless specifically explicitly instructed. They can easily introduce invalid formatting markers or hallucinate incorrect package configurations if left unchecked, requiring deep user technical oversight to correct.

### Question 2: How did you validate and verify that the AI-generated Markdown syntax was correct and rendered properly?
* **Validation Process:** Every section generated by the AI was cross-checked directly inside the GitHub interface via the visual Preview feature before committing. This allowed me to immediately spot when complex elements—such as nested link characters or code blocks—accidentally caused structural display errors, enabling manual line-by-line syntax updates to maintain code clarity.

### Question 3: Discuss the ethical implications of using AI for academic work, specifically regarding attribution and validation.
* **Ethical Considerations:** Utilizing AI as an interactive design or layout companion is highly productive, but failing to declare its use or blindly copying unverified content violates basic academic integrity guidelines. Proper attribution through an upfront AI Declaration ensures total transparency about the author's process, while rigorous manual validation guarantees that the student remains completely responsible for the final accuracy, quality, and logic of the work.
